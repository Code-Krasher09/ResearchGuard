# Overview

ResearchGuard is a self-correcting Retrieval-Augmented Generation (RAG)
system designed for scientific question answering.

Unlike traditional RAG systems, ResearchGuard introduces a verification
and repair stage that identifies unsupported claims generated by the
LLM, diagnoses potential retrieval failures, and iteratively improves
responses.

The project aims to address hallucination issues commonly observed in
large language model based systems.

---

# Problem Statement

Traditional RAG pipelines follow a straightforward process:

Query
→ Retrieval
→ Generation
→ Response

Although retrieval grounds the model in external information,
generated responses may still contain:

- Hallucinated facts
- Unsupported claims
- Contradictions
- Missing evidence
- Low-confidence statements

ResearchGuard attempts to mitigate these issues through claim-level
verification and adaptive retrieval repair mechanisms.
